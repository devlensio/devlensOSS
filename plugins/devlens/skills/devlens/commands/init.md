# /devlens init — set up DevLens for this repo

Bootstrap so the other subcommands work. Arguments: none.

## Steps
1. **MCP connected?** Call `list_analyzed_repos`. If the `mcp__plugin_devlens_devlens__*` tools aren't available, the DevLens MCP isn't registered — tell the user to enable this plugin or run `claude mcp add devlens -- npx -y @devlensio/cli mcp`, then retry.
2. **Provider configured (for summaries)?** Summaries are generated by an LLM the DevLens CLI is configured with. If summarization later fails for lack of a provider, the user configures it once via the CLI: `devlens init` (interactive) sets the provider/model/key. (This is a one-time CLI step; the MCP itself needs no setup.)
3. **Build the structural graph:** call `analyze` with this repo's path. This is structure only and safe to run.
4. **Offer summaries (ask first):** summaries (technical/business/security) cost LLM calls. **Ask the user** whether to generate them now; only if they agree, re-run `analyze` with summarization enabled.
5. **Confirm state:** `get_repo_overview` for this repo → show its commit/summary coverage so the user knows what's ready.

## Notes
- Summary-dependent subcommands (architecture, explain, security-analysis, summary) need step 4. Structure-only subcommands (find, impact, tech-debt, diagram) work after step 3.
- Re-run `init` after a big dependency or structure change if you want a fresh graph.
